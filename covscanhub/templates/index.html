Home page

Covscan is a tool that allows Red Hat users to run Coverity Scan on their packages
without logging on the Coverity server or installing their own Coverity server.
Users only need to download and install the covscan client and its requirements,
configure the client and acquire a valid Kerberos ticket (if not acquired already).
The usage of the covscan client is similar to brew/koji.
The client provides an easy to use command line interface.

Another benefit for covscan users in already included workarounds for Coverity bugs;
especially C++ packages are hard to scan with the default Coverity setup.

Please remember that we have only one shared Coverity license
and that's why only one package can be analyzed at a time in the whole Red Hat.
Please don't abuse this tool with multiple scans of scan expensive packages.
If we experience overload of this kind, we will blacklist the individuals.

Required setup before usage

  1. Make sure you have the latest version installed.
    • Install covscan-client and it's depenencies (kobo-client, python-krbV) from here - only RHEL-5/RHEL-6 rpms available
    • On Fedora, use rpmbuild --rebuild on the latest covscan.src.rpm from RHEL-6 repo to build covscan yourself (we'll provide Fedora packages later)
  2. Generate a kerberos ticket (kinit)

How to use it ?

  • Check available mock configs by covscan list-mock-configs, choose the best fit for your package
  • covscan mock-build --config=... [--security/--all] [--comment=...] /path/to/srpm
  • Wait for results (email notification)
  • Download the results (.xz tarball) from the covscan WebUI
  • Extract .xz, check the results, fix the defects :), share them with upstream

For additional information please check advanced usage page
