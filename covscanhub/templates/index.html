Home page

Welcome to web-based interface of covscan service.

Required setup before usage

  1. Install package covscan-client from this repo.
  2. Generate a kerberos ticket (with command kinit).
  3. For more info, check our wiki pages.

How to use it?

  • You can browse available mock configs with command covscan list-mock-configs, choose the best fit for your package.
  • Scan your package with command: covscan mock-build --config=... /path/to/srpm.
  • You can pick various analyzers (using option -a): check available with command covscan list-analyzers.
  • Wait for results (email notification).
  • Check the results in WebUI.

Differential scan

  • It may scan old and new version of a package and returns a list of added/fixed defects.
  • Both old and new packages can be provided as SRPM or by NVR (covscan downloads those from brew/koji).
  • Example: covscan version-diff-build -p errata --base-brew-build curl-7.29.0-19.el7 --srpm ./curl-7.29.0-20.el7.src.rpm.
  • File added.{err,js,html} contains a list of defects added in the new version.
  • File fixed.{err,js,html} contains a list of defects fixed in the new version.

Feel free to fill bugs or RFEs in GitLab.
